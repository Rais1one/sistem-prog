1. Что такое лямбда-выражение? В чём его преимущества перед обычной функцией?
Лямбда-выражение - это анонимная функция, которую можно определить прямо в месте использования. 
Преимущества: может захватывать переменные из окружающей области видимости, не требует отдельного объявления, делает код компактнее и локальнее.

2. Какие существуют способы захвата переменных в лямбдах?
Захват по значению [=], по ссылке [&], конкретных переменных [x] или [&x], захват this, а также комбинированные способы типа [=, &x] (все по значению, кроме x по ссылке).

3. Что такое std::function? Какие типы вызываемых объектов она может хранить?
std::function - это универсальная обёртка для любых вызываемых объектов. Может хранить обычные функции, лямбды, функторы и указатели на методы классов.

4. В чём разница между захватом по значению [=] и по ссылке [&]?
При захвате по значению создаётся копия переменной, изменение внутри лямбды не влияет на оригинал. При захвате по ссылке лямбда работает с оригиналом, изменения видны снаружи, но нужно гарантировать, что переменная существует во время вызова.

5. Когда использование std::function может быть неоптимальным с точки зрения производительности?
При частых вызовах в критических участках кода, в играх или real-time системах, так как std::function имеет накладные расходы на виртуальные вызовы и возможное динамическое выделение памяти.

6. Как проверить, хранит ли std::function вызываемый объект?
С помощью оператора приведения к bool: if (func) { ... } или сравнением с nullptr: if (func != nullptr) { ... }.

7. Что такое стирание типов (type erasure) в контексте std::function?
Это техника, при которой конкретный тип вызываемого объекта скрывается за единым интерфейсом. std::function запоминает реальный тип через шаблонный конструктор, но предоставляет унифицированный способ вызова.

8. Напишите лямбду, которая захватывает переменную factor по значению и умножает её на свой аргумент.
auto multiply = [factor](int x) { return factor * x; };

9. Как передать лямбду в функцию, ожидающую std::function?
Написать лямбду прямо в месте вызова: process([](int x) { cout << x; }) или передать заранее созданную лямбду.

10. В каких случаях лучше использовать шаблонный параметр вместо std::function?
Когда важна производительность и лямбда вызывается часто. Шаблонный параметр инлайнится и не имеет накладных расходов, в отличие от std::function.
